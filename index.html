05 — Capstone Project

Shiftora —
The Full Story

COMP 3078 · Capstone Project II · Team T38
Shiftora
Employee Scheduling & Labor Platform

A production-grade full-stack web application for managing employee schedules, tracking clock-in/out with breaks, recording tip pools, and generating daily & weekly labor reports — built for restaurant and retail managers.

Vite + React TypeScript Tailwind v4 shadcn/radix UI Express.js MongoDB JWT Auth bcrypt REST API
🎯
Project Vision
To replace the manual, error-prone scheduling systems used by small restaurant and retail businesses with a clean, role-based digital platform that saves managers hours per week and gives employees real-time visibility into their shifts and earnings.
👥
Target Users
Restaurant managers who need to schedule and track staff. Employees who need to clock in/out and view their shift history. Admins who need organization-level oversight of all employees and labor costs.
⚙️
System Architecture
Single-page React frontend communicating with a Node.js/Express REST API. MongoDB stores users, shifts, punches, and tips in separate collections. JWT tokens secure all protected routes. Vite powers the dev and production build pipeline.
📊
Key Outcomes
Fully functional three-role system deployed with live API. Automated daily and weekly wage/tip reports. Real-time clock-in tracking with break management. GitHub repository with full documentation and setup guide.

Project Summary

Shiftora is a full-stack scheduling, time-tracking, and labor reporting application built as the COMP 3078 Capstone Project II deliverable by Team T38 (Prabesh Shrestha & Moksh Chhetri). The application supports three user roles — manager, employee, and admin — each with a dedicated dashboard and permission set. Managers can create and delete shifts, view live employee status, enter daily tip pools, and generate labor reports. Employees can clock in and out, track breaks, and view assigned shifts. Admins have organization-level visibility across all users and data.

Business Requirements

AuthenticationSecure register/login with role-based JWT tokens. bcrypt password hashing. Persistent sessions via localStorage token storage.
Employee ManagementFull CRUD for employee records including name, role, department, hourly rate, join date, and employment status.
Shift SchedulingManagers can create shifts with start/end time and date, assign to employees, and delete as needed. Employees can view their upcoming schedule.
Punch TrackingClock-in, clock-out, break-start, and break-end endpoints with timestamps. Live manager dashboard showing current employee status.
Tip ManagementDaily tip pool entry by manager. Automated even split across all employees who worked that day, reflected in the daily report.
ReportsDaily report: wages, hours, tips per employee. Weekly overview: trailing 7-day labor summary. Both aggregated from live punch and shift data.

Project Plan

The project was executed across two capstone sprints. Sprint 1 (Jan–Mar 2026, COMP 3077) covered initial planning, database schema design, and backend API scaffolding. Sprint 2 (Feb–Apr 2026, COMP 3078) focused on full frontend build, feature completion, integration, testing, and deployment. Weekly team syncs were held via Google Meet with formal Minutes of Meeting documentation submitted as deliverables.

Requirements Analysis & Design

The data model consists of five MongoDB collections: users, employees, shifts, punches, and tips. The REST API exposes 16 endpoints grouped under /api/auth, /api/employees, /api/shifts, /api/punches, /api/tips, and /api/reports. Frontend state is managed via React hooks with an API client module centralizing all fetch calls and token injection.

System Implementation Highlights

  • JWT authentication with role-based route protection on both client and server
  • Fully responsive React UI using Tailwind v4 and shadcn/radix component library
  • Real-time punch status displayed on manager dashboard using polling
  • Report aggregation built server-side using MongoDB aggregation pipeline patterns
  • Environment-based configuration (server/.env and ./.env) supporting both local and Atlas MongoDB
  • Seeded demo accounts for immediate testing without registration
  • GitHub repository: github.com/prabesha1/Shiftora
